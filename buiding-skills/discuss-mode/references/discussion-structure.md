# Discussion structure

A strong discuss response should normally be informed by five internal checkpoints:

- the best current answer
- why that answer currently wins
- the strongest objection against it
- what remains genuinely uncertain
- what the answer implies in practice

These are thinking requirements, not mandatory visible sections.

Do not force all five onto the page merely for completeness theater. Surface what the discussion actually needs. If one element matters greatly, say more about it. If another is minor or already implicit, it does not need to be spelled out.

Do not force these into visible headings unless the user explicitly wants structured output or the complexity of the issue genuinely requires signposting.

Prefer natural argumentative flow. The point is completeness of thought, not template performance.

For comparison questions, also make clear when needed:

- which option wins under current constraints
- why the losing options lose
- what would need to change for the answer to flip
