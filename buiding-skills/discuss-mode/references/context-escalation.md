# Context escalation

## Escalate when a shallow answer would be irresponsible

Broaden repository reading when any of the following is true:

- the question concerns a strategic choice
- the user is comparing methods with meaningful cost or risk
- the project has a history of failed attempts that may matter
- the available summary files conflict with one another
- the question mentions "before", "previously", "historically", or similar cues
- the local evidence is too thin for a serious answer

## Escalation order

1. Start with `PROJECT.md`.
2. Read the most relevant recent `Experiment.md`.
3. Read additional experiments if the answer may depend on path history.
4. Read `OPERATIONS.md` when operational changes may affect interpretation.
5. Read directly implicated scripts, data notes, or outputs.

## Stop escalating when

- the answer is grounded enough to survive criticism
- further reading would only repeat already-known information
- the next missing piece is external rather than local
