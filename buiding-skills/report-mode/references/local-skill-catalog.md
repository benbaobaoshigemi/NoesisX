# Local skill catalog derived from awesome-ai-research-writing

These are the external writing-oriented skills explicitly highlighted in the source repository. Use them only when they materially improve the current report workflow.

## 20-ml-paper-writing
For end-to-end paper drafting, conference-template migration, citation-aware paper writing, and structured paper sections.
Use when the task is close to full paper drafting or conference-paper formatting.

## humanizer
For identifying and removing common AI-writing residue and making the prose feel more natural.
Use mainly in an explicit polishing pass.

## docx
For `.docx` creation, editing, template filling, and tracked-change workflows.
Use when the requested deliverable must land in Word-oriented format or template.

## doc-coauthoring
For phased drafting and refinement with section-by-section collaboration.
Use when the user wants a multi-stage, collaborative drafting process rather than a single pass.

## canvas-design
For concept figures and single-page design artifacts.
Usually out of scope for pure report writing, but relevant if the deliverable explicitly includes a concept diagram.

## Rule

Mention a local skill in the subagent brief only if it is clearly relevant and locally available. Otherwise proceed with the bundled local prompt references in this skill.
