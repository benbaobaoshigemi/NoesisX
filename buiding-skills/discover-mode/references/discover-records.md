# discover records

Discover mode always leaves a record trail.

## mandatory updates when discover stops

### chat summary

The chat summary must state:

- what was explored
- what changed direction
- what was abandoned or deprioritized
- what boundary caused the stop
- what remains unresolved

### project.md

Update the project-level memory with:

- current project question
- strongest active interpretation
- abandoned or weakened paths
- current unresolved questions
- next high-value direction if work resumes

### experiment.md

Update the relevant experiment record with:

- the scientific objective of the discover run
- materials inspected
- methods or comparisons used
- findings and interpretations
- the stop boundary reached

### operations.md

Update operations with non-experimental work such as:

- file movement
- directory cleanup
- exports
- visualization generation
- formatting
- encoding fixes
- environment fixes
- any other workspace modifications

## direction-shift note

Whenever discover changes direction, record:

- previous line of inquiry
- new line of inquiry
- why the shift happened
- what evidence forced or justified it

## file writing discipline

All file-written text must be de-dialogized.

Do not write files as if responding to the current user.
Do not preserve the current chat's interactional tone.
Do not rely on shared conversational context.
