# Writer Topologies

Choose the simplest topology that satisfies the task.

## Single

Use when:

- one-pass polishing is enough
- translation is direct
- structure optimization is a single bounded task

## Cascade

Use when:

- the user explicitly wants drafting then polishing
- a document should first be written, then reviewed from another posture
- translation should be followed by a style cleanup or compliance pass

## Parallel

Use when:

- the user wants multiple candidate versions
- multiple sections can be written independently
- reviewer and author-facing versions should be generated side by side

## Rule

- Single is the default.
- Cascade and parallel must be justified by the task, not by habit.
