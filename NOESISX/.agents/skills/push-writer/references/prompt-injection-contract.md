# Prompt Injection Contract

`push-writer` must convert local prompt files into actual writer prompt content.

## Forbidden

- “See attached prompt file.”
- “Please read these references yourself.”
- “Go choose one from the references directory.”

## Required

The final writer brief must explicitly embed:

- user objective
- deliverable type
- target language
- factual boundary
- scope boundary
- output format
- selected prompt rules from local files
- formal style constraints
- rendering and layout obligations
- table and formula requirements
- any required downstream export constraint

## Injection shape

When injecting local prompt content, preserve the useful rules but adapt wording to the current task.

Use this structure:

1. task
2. selected mode
3. non-negotiable boundaries
4. formal style charter
5. injected prompt rules
6. output and rendering requirements
7. output expectation

## Rule

Injection is not optional. If local prompt content was selected, it must appear in the writer brief body.
